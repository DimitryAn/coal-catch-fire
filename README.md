## Запуск сервиса 
Для запуска нашего web-сервиса необходимо проделать следующие шаги.
1) У вас должен быть установлен Docker, в случае отсутствия можно скачать с офицального сайта по ссылке - https://www.docker.com/get-started/
2) Необходимо склонировать проект с данного репозитория. Допустим, по https
```bash
git clone https://github.com/DimitryAn/coal-catch-fire.git
```
3) После загрузки проекта запускаем с помощью команды
```bash
docker compose up --build
```
4) Чтобы проверить работу, необходимо открыть страницу  http://localhost:8000/ в вашем браузере 
5) Для завершения работы приложения выполните в термениле комбинацию 
```bash
Ctrl + C
```
6) Завершение работы всех контейнеров
```bash
docker compose down
```

## Руководство по использованию приложения

1. Добавьте файлы temperatures.csv, supplies.csv, weather.csv в окне "Загрузите CSV файлы".
Для этого можно нажать по области, выделенной пунктиром, и в открывшемся окне выбрать запрашиваемые файлы или перетащить файлы в это окно. 

    **Необходимо сразу добавить все 3 файла. В окне должно отображаться "Выбрано файлов: 3"**

    После чего нажать кнопку "Загрузить в базу"

    Загруженные файлы можно просмотреть в окне "Просмотр данных". Для этого нужно выбрать в списке таблицу, которую вы хотите просмотреть, после чего надать кнопку "Загрузить". Также можно 

2. В окне "Таблица самовозгораний" появится таблица со столбцами - предсказанными датами и строками - номерами складов. В ячейках находятся номера штабелей, которые загорятся в соответствующий день и на соответствующем складе.
   
3. Теперь можно загрузить файл fires.csv в окно "Загрузите CSV-файл с реальными пожарами", загрузка файла происходит аналогично п.1.

4. В окне "Метрики" появятся метрики, посчитанные используя реальные данные. 
   
   В окне "Таблица истинных возгораний" появится таблица самовозгораний, совпадающая по структуре с таблицей в п.2, но с данными о реальных пожарах

5. После вывода метрик и таблицы модель дообучится.
